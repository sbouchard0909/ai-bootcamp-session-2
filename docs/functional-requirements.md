# Functional Requirements

## Overview

This document outlines the core functional requirements for the task management application.

## Requirements

### FR-1: Add Tasks

Users must be able to create new tasks.

- A user can enter a task title and submit it to add it to the task list.
- Newly added tasks appear in the task list immediately.

### FR-2: Edit Tasks

Users must be able to modify existing tasks.

- A user can update the title of any existing task.
- Changes are reflected in the task list immediately upon saving.

### FR-3: Reorder Tasks

Users must be able to change the order of tasks in the list.

- A user can move a task up or down relative to other tasks.
- The updated order persists after the action is performed.

### FR-4: Add a Due Date to Tasks

Users must be able to associate a due date with a task.

- A user can set or update a due date on any task.
- The due date is displayed alongside the task in the task list.

### FR-5: Delete Tasks

Users must be able to remove tasks from the list.

- A user can delete any existing task.
- The task is removed from the task list immediately upon deletion.
